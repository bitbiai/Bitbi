# AI Cost Route Inventory

Date: 2026-05-16

Scope: provider-cost route inventory for Phase 3.1 plus Phase 3.3 operation-registry baseline, Phase 3.4 member personal image pilot, Phase 3.5 member music cost decomposition, Phase 3.6 member music gateway migration, Phase 3.7 replay/cover/finalization/cleanup hardening for member image/music, Phase 3.8 member video gateway migration, Phase 3.9 enforcement guard/known-gap baseline, Phase 4.1 admin/platform budget policy design, Phase 4.2 admin/platform budget policy helper contract, Phase 4.3 charged Admin BFL image-test hardening, Phase 4.4 read-only Admin/Platform budget evidence reporting, and Phase 4.5 admin async video job budget enforcement. This does not approve production/live billing readiness and does not claim full AI Cost Gateway coverage.

Legend:

- Idempotency: required, partial, recommended, absent, not applicable.
- Reservation: pre-provider durable reservation or equivalent provider-call suppression.
- Replay: successful same-key result replay/cache that avoids another provider call.
- Gap severity: P0/P1/P2/P3 for future work priority, not a production readiness approval.

## Summary

Highest-risk current gaps:

- Member no-organization `/api/ai/generate-image` is now the Phase 3.4 pilot plus Phase 3.7 hardening. It requires `Idempotency-Key`, reserves credits in `member_ai_usage_attempts` before provider execution, suppresses same-key duplicate provider calls, replays safe temporary image metadata while available, returns replay-unavailable without re-executing providers or double-debiting, debits exactly once after provider success, and releases/no-charges on provider failure.
- Phase 3.6 migrates member music into one parent `member_ai_usage_attempts` reservation. Phase 3.7 hardens it with replay-unavailable behavior, cover status writeback, finalization safety, and member attempt cleanup. It requires `Idempotency-Key`, reserves before lyrics/audio/cover work, suppresses same-key duplicate provider calls, replays safe persisted-asset metadata when available, debits exactly once after audio persistence, and releases/no-charges on lyrics/audio provider failure.
- Phase 3.8 migrates member video into one parent `member_ai_usage_attempts` reservation. It requires `Idempotency-Key`, reserves before PixVerse/HappyHorse provider work and remote output ingest, suppresses same-key duplicate provider calls, replays safe durable saved-asset metadata when available, returns replay-unavailable without provider re-execution or double debit when the saved object is missing, and debits exactly once after durable video persistence.
- Phase 3.9 adds `config/ai-cost-policy-baseline.json` and makes the local AI cost policy guard fail on unbaselined provider-cost route/source drift, duplicate registry or baseline ids, missing baseline references, and member image/music/video gateway regression.
- Phase 4.1 maps remaining admin/platform/internal/OpenClaw gaps to target budget scopes: `platform_admin_lab_budget`, `admin_org_credit_account`, `openclaw_news_pulse_budget`, `internal_ai_worker_caller_enforced`, and related reviewed exception scopes.
- Phase 4.2 adds pure helper contracts and tests for those scopes: kill-switch metadata validation, deterministic fingerprints, safe audit fields, and plan status classification.
- Phase 4.3 imports that helper only from the existing charged Admin image-test branch. It records safe `admin_org_credit_account` budget plan/audit metadata, deterministic policy fingerprint, and future kill-switch target metadata while preserving selected-organization credit debits, required `Idempotency-Key`, and no-charge-on-provider-failure behavior.
- Phase 4.4 adds a read-only budget evidence helper, admin endpoint, and local script that report covered, baselined, unmetered, caller-enforced, and missing-runtime-enforcement flows without changing runtime enforcement or migrating routes.
- Phase 4.5 covers only admin async video jobs. It stores sanitized `platform_admin_lab_budget` metadata on job rows, includes bounded queue budget summaries, verifies job budget state before internal video task create/poll calls, and suppresses duplicate provider task creation after an unresolved create attempt. Broad Admin AI, OpenClaw/News Pulse, platform/background AI, and internal AI Worker routes globally remain unmigrated.
- Admin AI Lab provider routes are admin-only but mostly uncharged and not uniformly idempotent, reserved, or replayable.
- News Pulse visual generation can spend AI provider cost from OpenClaw ingest waitUntil and scheduled backfill outside member/org billing scope.

## Current Cross-Cutting Evidence

- `workers/auth/src/app/route-policy.js` records billing metadata for member image/text/music/video routes. Member image, music, and video are required-idempotency gateway routes for their migrated member paths; org text/image still use `ai_usage_attempts`.
- `workers/auth/src/lib/ai-usage-policy.js` requires idempotency and creates `ai_usage_attempts` for organization context and `member_ai_usage_attempts` for the migrated member image/music/video paths.
- `workers/auth/src/routes/ai/text-generate.js` uses org-only reservation/replay and delegates provider execution to the HMAC-protected AI worker.
- `workers/auth/src/routes/ai/images-write.js` implements both legacy/member mode and org-scoped attempt mode in one handler.
- `workers/auth/src/routes/ai/music-generate.js` and `workers/auth/src/routes/ai/video-generate.js` are member personal-credit routes and reject organization context.
- `workers/auth/src/routes/admin-ai.js` contains admin-only proxy routes plus Phase 4.3 charged admin image-test handling and Phase 4.5 admin async video job create handling. The charged image branch records safe `admin_org_credit_account` metadata; admin video jobs record `platform_admin_lab_budget` metadata; the unpriced image branch and other Admin AI routes remain baselined gaps.
- `workers/auth/src/lib/admin-platform-budget-evidence.js` and `GET /api/admin/ai/budget-evidence` expose Phase 4.4 sanitized read-only evidence updated for Phase 4.5 admin video coverage. The report verdict remains blocked and performs no provider calls, billing mutations, credit mutations, D1/R2 writes, Stripe calls, or route migrations.
- `workers/auth/src/lib/member-music-cover.js` and `workers/auth/src/lib/news-pulse-visuals.js` contain direct `env.AI.run` calls outside the member/org credit gateway.
- `workers/auth/src/lib/admin-ai-proxy.js` signs Auth-to-AI Worker proxy calls; the provider cost is incurred by the internal AI Worker route it calls, not by the proxy helper itself.
- `workers/ai/src/lib/invoke-ai.js` contains the shared AI Worker `env.AI.run` implementations for text, image, embeddings, and music.
- `workers/ai/src/lib/invoke-ai-video.js` contains the AI Worker video provider execution paths, including Workers AI, AI Gateway, Vidu fallback, provider task create, and provider task poll helpers.
