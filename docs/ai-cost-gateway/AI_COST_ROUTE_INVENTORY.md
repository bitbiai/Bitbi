# AI Cost Route Inventory

Date: 2026-05-15

Scope: provider-cost route inventory for Phase 3.1 plus Phase 3.3 operation-registry baseline. This is evidence and design only. It does not change runtime behavior, call AI providers, mutate billing, or approve production/live billing readiness.

Legend:

- Idempotency: required, partial, recommended, absent, not applicable.
- Reservation: pre-provider durable reservation or equivalent provider-call suppression.
- Replay: successful same-key result replay/cache that avoids another provider call.
- Gap severity: P0/P1/P2/P3 for future work priority, not a production readiness approval.

## Summary

Highest-risk current gaps:

- Member `/api/ai/generate-video` and `/api/ai/generate-music` only recommend `Idempotency-Key`, check credits before provider calls, and debit after success. They do not reserve credits before provider execution, suppress duplicate provider execution, or replay successful results.
- Member no-organization `/api/ai/generate-image` also uses optional member idempotency and post-provider debit. It can still spend provider cost on duplicate or concurrent requests before the member ledger rejects/records a duplicate debit.
- Member music can trigger multiple provider-cost steps: optional lyrics text generation, music generation, and background cover image generation.
- Admin AI Lab provider routes are admin-only but mostly uncharged and not uniformly idempotent, reserved, or replayable.
- News Pulse visual generation can spend AI provider cost from OpenClaw ingest waitUntil and scheduled backfill outside member/org billing scope.

## Phase 3.3 Registry Baseline

`workers/auth/src/lib/ai-cost-operations.js` is the current report-only operation registry. It records 30 known AI cost operations, including 29 provider-cost operations and one not-applicable personal member text baseline. The registry validates each target operation config through `normalizeAiCostOperationConfig()` and records current enforcement status separately from target policy.

Current registry summary:

- member operations: 6
- organization operations: 2
- admin/platform operations: 22
- current missing mandatory idempotency: 4
- current missing reservation: 4
- current no replay: 4
- platform budget review operations: 3
- highest-risk operations: `member.image.generate`, `member.music.generate`, `member.music.lyrics.generate`, `member.music.cover.generate`, `member.video.generate`, `internal.music.generate`

The registry is not imported by live routes yet. It is used by `npm run check:ai-cost-policy` and `npm run test:ai-cost-operations` only.

## Current Cross-Cutting Evidence

- `workers/auth/src/app/route-policy.js` records billing metadata for member image/text/music/video routes. Music and video explicitly describe idempotency as recommended, not required.
- `workers/auth/src/lib/ai-usage-policy.js` requires idempotency and creates `ai_usage_attempts` only for organization context. Member mode accepts `Idempotency-Key` when supplied but does not reserve credits before provider execution.
- `workers/auth/src/routes/ai/text-generate.js` uses org-only reservation/replay and delegates provider execution to the HMAC-protected AI worker.
- `workers/auth/src/routes/ai/images-write.js` implements both legacy/member mode and org-scoped attempt mode in one handler.
- `workers/auth/src/routes/ai/music-generate.js` and `workers/auth/src/routes/ai/video-generate.js` are member personal-credit routes and reject organization context.
- `workers/auth/src/routes/admin-ai.js` contains admin-only proxy routes plus charged admin image test handling for priced image models.
- `workers/auth/src/lib/member-music-cover.js` and `workers/auth/src/lib/news-pulse-visuals.js` contain direct `env.AI.run` calls outside the member/org credit gateway.
- `workers/auth/src/lib/admin-ai-proxy.js` signs Auth-to-AI Worker proxy calls; the provider cost is incurred by the internal AI Worker route it calls, not by the proxy helper itself.
- `workers/ai/src/lib/invoke-ai.js` contains the shared AI Worker `env.AI.run` implementations for text, image, embeddings, and music.
- `workers/ai/src/lib/invoke-ai-video.js` contains the AI Worker video provider execution paths, including Workers AI, AI Gateway, Vidu fallback, provider task create, and provider task poll helpers.

## Verification Needed

Needs verification before runtime gateway work:

- Whether member optional idempotency currently prevents duplicate debits but not duplicate provider calls for every member image/music/video edge case.
- Whether async admin video queue retries can create duplicate provider tasks if the provider task create succeeds but the response is lost before `provider_task_id` persists.
- Whether News Pulse generated visuals should be budgeted separately from member/org credits or put under a platform quota.
- Whether generated music covers should be bundled into music credits, separately metered, or disabled when cover generation fails repeatedly.
