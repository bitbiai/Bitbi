# Contributing to Bitbi

Thanks for your interest in contributing to Bitbi.

Please read these guidelines before suggesting changes or submitting contributions.

## Before making bigger changes
For major changes, new features, architecture changes, or larger refactors, please open a Discussion or Issue first before starting work.

## General expectations
- Keep changes focused and small when possible.
- Respect the existing project structure and architecture.
- Do not introduce new dependencies unless they are really necessary.
- Avoid breaking existing functionality, especially authentication, member features, and mobile behavior.
- Write clear descriptions of what changed and why.

## Bug reports
If you found a bug, please include:
- where it happens
- steps to reproduce it
- what you expected
- what happened instead
- screenshots if helpful

## Feature requests
If you want to suggest a feature, please explain:
- the problem you want to solve
- your proposed solution
- why it would improve Bitbi

## Pull requests
If you submit a pull request:
- keep it as small and focused as possible
- describe the change clearly
- mention any affected pages, features, or backend routes
- include screenshots for UI changes when useful

Thank you for helping improve Bitbi.
